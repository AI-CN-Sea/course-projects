# 测试运行说明

本目录用于 PL/0 编译器扩展功能的本地检查。`pl0.c`、`pl0.h` 是最终源码，`tests` 文件夹中集中存放测试源程序，`output` 文件夹用于保存每个测试程序运行后生成的文件。编译器会按照输入文件名建立对应输出目录，例如输入 `tests\test_array2.txt` 后，结果会保存到 `output\test_array2` 中。

## 运行方法

在 VS Code 终端进入本目录后，先编译源码：

```powershell
gcc -Wall -Wextra pl0.c -o pl0.exe
```

然后运行编译器：

```powershell
.\pl0.exe
```

程序提示 `Input pl/0 file?` 时输入测试文件路径，例如：

```text
tests\final_input.txt
```

后面两个问题分别表示是否在终端中列出目标代码、是否列出符号表。综合用例建议对 `tests\final_input.txt` 输入 `Y` 和 `Y`，这样终端中可以同时体现源程序、P-code、符号表和最终输出。普通单项测试可以输入 `N` 和 `N`，重点查看最终输出即可。

## 输出文件说明

每个测试运行后，编译器会在 `output` 下建立对应的子目录。常见文件含义如下：

| 文件名 | 含义 |
| --- | --- |
| `fa.tmp` | 生成的 P-code 目标代码 |
| `fa1.tmp` | 源程序行与代码地址对应信息 |
| `fa2.tmp` | 程序解释执行后的输出结果 |
| `fas.tmp` | 符号表输出结果 |

`fas.tmp` 是否有内容取决于运行时是否选择输出符号表，以及程序是否已经成功分析到能够输出符号表的位置。例如 `test_error_comment.txt` 是注释未闭合错误，错误发生在词法分析阶段；如果运行时又选择不输出符号表，则 `output\test_error_comment\fas.tmp` 为空是正常现象。

## 测试设计目的

测试用例用于覆盖题目 11.1 的核心要求，并验证各个扩展功能既可以单独运行，也可以组合运行。单项测试用于定位某一个语法扩展是否正确，综合测试用于验证多个扩展之间没有冲突，错误测试用于验证编译器能够发现常见错误并给出错误编号。

`final_input.txt` 是综合测试，用于同时覆盖一维数组、IF-THEN-ELSE、REPEAT-UNTIL、带传值参数过程和注释处理。

数组相关测试包括 `test_array.txt`、`test_array2.txt` 和 `test_read_array.txt`。其中 `test_array.txt` 验证数组基本读写与求和，`test_array2.txt` 验证非 0/1 下界数组的地址计算，`test_read_array.txt` 验证 `read(a(i))` 形式能否把输入值写入数组元素。

条件语句测试包括 `test_if_true.txt` 和 `test_if_false.txt`，分别验证 THEN 分支和 ELSE 分支，避免漏掉 ELSE 跳转地址返填或跳转目标错误。

循环测试使用 `test_repeat.txt` 验证 REPEAT-UNTIL。该结构与 WHILE 的主要区别是循环体至少执行一次，因此测试中通过连续输出 1 到 5 来观察循环体执行顺序和终止条件是否正确。

过程参数测试包括 `test_param.txt`、`test_param2.txt` 和 `test_param_value.txt`。`test_param.txt` 验证普通传值参数过程，`test_param2.txt` 验证带参数递归调用，`test_param_value.txt` 验证形参修改不会反向改变实参变量。

注释测试使用 `test_comment.txt` 验证 `{ }` 注释能在词法分析阶段被跳过，不影响后续语法分析和代码生成。

错误测试包括 `test_error_param.txt`、`test_error_repeat.txt`、`test_error_array_bound.txt`、`test_error_comment.txt` 和 `test_error_period.txt`，分别对应实参与形参数量不一致、REPEAT 语句缺少 UNTIL、数组声明边界错误、注释未闭合以及程序结束符错误。
